<테이블태그>

테이블 태그는 과거에는 사이트의 모든 레이아웃 구성을 담당했었지만
지금은 게시판 / 회원가입 화면 / 표의 형태의 디자인을 구성할 때 사용이 되는 형태로 취급
<table></table> 테이블 표 자체를 감싸는 태그
<caption></caption> 테이블 표의 제목을 나타내는 태그 (생략 가능 / 쓰는 걸 권장)

<colgroup></colgroup> 테이블 표의 열(col) 부분의 너비(가로 사이즈)를 담당하는 태그 
<col width="사이즈값"> 테이블 표의 열(col) 한 줄씩 담당하는 태그 (실제 한 개의 열 가로크기를 지정하는 태그)
         
<tr></tr>  테이블 표의 한 줄(행 row)을 표현하는 태그  table row
<th></th>  테이블 표의 한 줄 안에 들어가는 한 칸 을 담당하는 태그 (표 안에서 소제목을 표현할떄) table header
<td></td>  테이블 표의 한 줄안에 들어가는 한 칸을 담당하는 태그 (표 안에서 실질적인 내용, 데이터) table data


* css reset 할때 추가될 테이블태그에 지정해야 하는 속성 
  table {border-collapse: collapse;}
- border-collapse 속성은 테이블 안에 있는 칸과 칸 사이 간격 지정하는 속성
  boeder-collapse 속성은 기본값이 separate (칸과 칸 사이 분리해서 표시)
  border-collapse 속성을 collapse (칸과 칸 사이가 붙어있는 상태로 표시되기 때문에 css reset 시 해당 값을 지정함)
  table 태그에 있는 칸에 한해서만 테두리선이 이중으로 안뜸


* col부분에 가로값을 지정할 떄 %로 사용하는 경우는 
  테이블태그에 가로값이 있어야 그 사이즈 기준으로 열들의 % 수치가 결정된다. 
  col부분에 가로값을 px로 지정하는 경우는
  테이블태그의 가로값이 없어도 col부분에 지정한 가로크기 px 수치값을 합산해서 테이블의 총 가로사이즈가 결정됨

* 가로값,세로값 지정가능 / 여백 4방향 지정가능  / 테이블 태그 다음에 작성하는 태그는 줄바꿈 

* th 태그 안에 있는 내용들은 기본적으로 가운데 정렬이 반영되서 나옴
* td 태그 안에 있는 내용들은 기본적으로 세로 방향만 가운데로 정렬되서 나옴. 가로 방향은 text-aline:center 적용할 것
* 표 안에 칸을 담당하는 태그들은 기본적으로 옆으로 표시된다.
